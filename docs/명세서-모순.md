# 명세서 간 모순 / Supersede 트래킹

> AGENTS.md §1 (우선순위) 와 §5 #6 (모순 보고) 에 따라, 명세서 4종 (요구·기능·기술·SDD) 간 또는 정본 명세서와 후속 CEO 정책·ADR 간의 모순을 본 문서에 등록한다. **본 문서는 정본의 일부가 아니라 추적 기록**이며, 실제 결정은 우선순위 사슬 (이슈 본문 > CEO 브리프 > SDD > 기술 > 기능 > 요구) 과 명시적 ADR 로 봉인된다.
>
> 신규 모순 발견 시 표 하단에 row 1건씩 append 하고, 해당 항목이 ADR 또는 후속 명세 리비전으로 해소되면 `Status` 를 `Resolved` 로 옮긴다.

## 모순 목록

### CFLT-001 — 사용자 인증: 로그인 필수 vs 비회원 사전검토 허용

- **Status**: Superseded (CEO 정책 CMP-557 / ADR-0003)
- **발견 일자**: 2026-05-29 (CMP-558)
- **관련 이슈**: CMP-557 (사용자 식별 정책 결정) · CMP-558 (정책 문서화 트랙 A)
- **충돌 정본**:
  - `docs/명세서/02. 집핀_기능명세서_v1.0.docx` §사용자 흐름 1번 행 — `로그인 | 소셜 OAuth (카카오/구글/네이버) 로그인 필수` (`docs/_extracted/02_집핀_기능명세서_v10.txt:24`)
  - 동 명세서 시스템 가정 — `사용자 인증: 소셜 OAuth 필수. 비회원 사전검토 불가` (`docs/_extracted/02_집핀_기능명세서_v10.txt:38`)
  - 동 명세서 `AUTH-001  소셜 OAuth 로그인` — `사용자는 카카오·구글·네이버 계정으로 로그인해야 서비스를 이용할 수 있다.` (`docs/_extracted/02_집핀_기능명세서_v10.txt:43-44`)
  - `docs/명세서/03. 집핀_기술명세서_v1.6.docx` §API 7.1 — `GET /auth/{provider}` 가 진입 전제로 기술됨 (`docs/_extracted/03_집핀_기술명세서_v16.txt:136`)
  - `docs/명세서/04. 집핀_소프트웨어설계문서_v1.9.docx` §AUTH 모듈 책임 — `소셜 OAuth 기반 사용자 식별과 JWT 세션 수명주기` 가 전체 사용자 진입의 단독 전제 (`docs/_extracted/04_집핀_소프트웨어설계문서_v19.txt:127`)
- **새 정본**: `AGENTS.md §4.7` · `docs/adr/0003-anon-user-and-sso.md`
- **요지**: CEO 정책은 **비회원 사전검토 허용 + 전환 시점(상담·리드·리포트 저장) OAuth 간편가입 의무** 로 변경된다. 도면 업로드·마스킹·1차 AI 판단·리포트 미리보기까지는 익명 세션으로 진행 가능하며, 전환 시점에서만 OAuth 게이트가 켜진다.
- **영향 모듈**: AUTH (anonymous_users / external_sso_accounts / terms_consents 신설), INPUT/CHAT/REPORT (익명 세션 컨텍스트 전달 경로), Frontend (전환 시점 인터셉트 UX).
- **후속 액션**:
  - 명세서 4종 다음 리비전(요구사항 v0.3+, 기능 v1.1+, 기술 v1.7+, SDD v2.0+) 에서 본 충돌 절을 직접 수정. 본 문서가 정본 리비전을 대체하지 않는다.
  - 기능명세서 `AUTH-001` 은 **`AUTH-001-A 익명 세션 진입` + `AUTH-001-B 전환 시점 OAuth 간편가입`** 두 항목으로 분리되도록 권고.

### CFLT-002 — 사용자 비밀번호 컬럼: SDD `비밀번호는 보관하지 않는다` vs 명세서 4종 어디에도 `password` 필드 부재

- **Status**: No Conflict (재확인) — 본 줄은 향후 모순으로 보이지 않도록 사전 등록.
- **발견 일자**: 2026-05-29 (CMP-558)
- **요지**: SDD §AUTH 데이터 계약은 `비밀번호는 보관하지 않는다` 를 명문화 (`docs/_extracted/04_집핀_소프트웨어설계문서_v19.txt:130`). 기술명세서 `users` 테이블도 `oauth_provider`·`oauth_sub` 만 둔다 (`docs/_extracted/03_집핀_기술명세서_v16.txt:236-237`). **CEO 정책 CMP-557 의 “password 컬럼 영구 금지” 는 기존 정본과 정합** — supersede 가 아니라 강화.
- **후속 액션**: ADR-0003 의 모델 메타데이터 테스트 가드 (CMP-557 트랙 B/C/D 의 모델 PR) 가 본 정합을 회귀로 잠근다.

### CFLT-003 — OAuth provider 식별: 기술명세서 `VARCHAR(20)` vs CEO 정책 `Postgres ENUM external_sso_provider`

- **Status**: Open (정책 우위 — 다음 기술명세서 리비전에서 정렬)
- **발견 일자**: 2026-05-29 (CMP-558)
- **충돌 정본**:
  - `docs/명세서/03. 집핀_기술명세서_v1.6.docx` §users 테이블 — `oauth_provider | VARCHAR(20) | NOT NULL | kakao | google | naver` (`docs/_extracted/03_집핀_기술명세서_v16.txt:236`)
- **새 정본**: `AGENTS.md §4.7` 6번 · `docs/adr/0003-anon-user-and-sso.md`
- **요지**: CEO 정책은 provider 를 **Postgres ENUM** 으로 봉인한다. 신규 provider 추가는 ADR + 마이그레이션이 필요하다 (런타임 string drift 차단). 기술명세서 v1.6 의 `VARCHAR(20)` 표기는 ADR-0003 적용 시 ENUM 으로 정렬한다.
- **후속 액션**: 기술명세서 v1.7+ 에서 컬럼 타입을 `external_sso_provider` ENUM 으로 수정. AUTH 모듈 마이그레이션 PR (CMP-557 트랙 B) 에 ENUM 정의 포함.

### CFLT-004 — Kakao Sync 약관 동의 처리

- **Status**: Open (정책 신설 — 명세서 4종 미수록 사항)
- **발견 일자**: 2026-05-29 (CMP-558)
- **요지**: 기존 명세서는 Kakao Sync 의 약관 동의 source 분리 저장을 다루지 않는다. CEO 정책은 Kakao Sync 약관 동의 source 를 우리 내부 약관 동의와 **분리해 저장** 하도록 요구한다 (`AGENTS.md §4.7` 5번). 기능·기술명세서 다음 리비전에서 `terms_consents.source` 컬럼과 Kakao 콜백 처리 순서를 반영해야 한다.
- **후속 액션**: AUTH 모듈 모델 정의 (CMP-557 트랙 B) 와 콜백 핸들러 (CMP-557 트랙 C) 에서 분리 저장 가드.

### CFLT-005 — 비회원 식별자 보관소: `localStorage` (CEO) vs 명세서 미정의

- **Status**: Open (정책 신설)
- **발견 일자**: 2026-05-29 (CMP-558)
- **요지**: 비회원 식별자는 서버가 발급한 UUID v4 를 `localStorage.jippin_anonymous_user_id` 키에 보관한다 (`AGENTS.md §4.7` 7번). 명세서 4종은 비회원 식별자를 다루지 않는다. 가입 성공 시 `anonymous_users.converted_user_id` / `converted_at` 으로 claim.
- **후속 액션**: 기능명세서 v1.1+ 에 “익명 세션 식별자 흐름” 절 추가. Frontend (Next.js) 미들웨어가 `localStorage.jippin_anonymous_user_id` 를 읽어 **`x-jippin-anon-id: <uuid>`** 헤더로 백엔드에 전달한다. 헤더명은 `apps/api/.env.example` 의 `ANON_SESSION_HEADER` 가 정본이며, 환경별 오버라이드가 필요하면 그 변수만 갱신한다. 헤더 이름 자체가 갱신될 경우 본 row 와 ADR-0003 §5 봉인 표를 함께 업데이트.

## 변경 절차

- 본 문서는 누구나 PR 로 row 를 append 할 수 있다 (gitmoji `📝 docs:`).
- **모순 해소 PR** (명세서 다음 리비전 또는 ADR 발행) 머지 시 해당 row 의 `Status` 를 `Resolved` 또는 `Superseded` 로 옮기고, 해소 ADR/리비전 링크를 본문에 기입한다.
- **ADR 발행이 모순 해소의 정본 경로** 다. 본 문서만 갱신하고 ADR 을 생략하면 안 된다.
